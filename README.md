## 1. RESUMO

Este repositório apresenta a organização e sistematização de materiais didáticos voltados ao estudo de **Algoritmos e Lógica de Programação**. O projeto utiliza o método de curadoria de fontes híbridas, integrando videoaulas do canal *Curso em Vídeo* e apostilas de instituições de ensino superior (USP, UFSM, IFC, SENAI e CAPES). ## 9. ANÁLISE DE RESULTADOS E TROUBLESHOOTING (CICATRIZES)

Nesta seção, documento os desafios encontrados durante a interação com as IAs e como os prompts foram refinados para obter respostas mais precisas:

- **Problema de Abstração:** Ao solicitar uma explicação sobre "Tabela Verdade" baseada na apostila da USP, a IA retornou uma resposta puramente matemática, o que dificultou a compreensão inicial.
- **Ajuste de Rota:** Refinei o prompt pedindo uma analogia com situações do cotidiano (como login em redes sociais) e solicitando que a IA agisse como um professor de ensino fundamental.
- **Refinamento no NotebookLM:** O cruzamento das 5 fontes gerou, inicialmente, respostas que misturavam termos técnicos avançados com explicações simples do Scratch. Precisei instruir a IA a "separar por camadas", apresentando primeiro a regra técnica e depois a aplicação prática.
- **Resultado Final:** As respostas tornaram-se mais assertivas após a imposição de restrições de contexto, focando exclusivamente no cronograma das aulas assistidas.

## 10. 🎯 MINIGUIA DE ESTUDO (ENTREGA FINAL)

### 10.1 Resumos Estruturados

---

#### 📌 Algoritmos — O que são?

Um **algoritmo** é uma sequência finita e ordenada de passos lógicos para resolver um problema. Todo algoritmo possui três características essenciais:

- **Entrada:** dados fornecidos pelo usuário ou pelo sistema;
- **Processamento:** operações realizadas sobre os dados;
- **Saída:** resultado gerado após o processamento.

**Exemplo do cotidiano:** a receita de um bolo é um algoritmo — tem ingredientes (entrada), modo de preparo (processamento) e o bolo pronto (saída).

---

#### 📌 Variáveis

Uma **variável** é um espaço reservado na memória do computador para armazenar um valor que pode mudar durante a execução do programa.

| Conceito | Descrição | Exemplo |
|----------|-----------|---------|
| Nome | Identificador da variável | `idade`, `nome`, `salario` |
| Tipo | Natureza do dado armazenado | inteiro, real, texto, lógico |
| Valor | Conteúdo armazenado | `25`, `"Maria"`, `true` |

**Regras para nomear variáveis:**
- Não usar espaços (use `_` ou camelCase);
- Não começar com número;
- Não usar caracteres especiais (ã, ç, é...).

---

#### 📌 Operadores Lógicos

Os **operadores lógicos** combinam condições para produzir um resultado verdadeiro ou falso.

| Operador | Símbolo | Resultado |
|----------|---------|-----------|
| E (AND) | `&&` | Verdadeiro **somente** se ambas as condições forem verdadeiras |
| OU (OR) | `\|\|` | Verdadeiro se **ao menos uma** condição for verdadeira |
| NÃO (NOT) | `!` | Inverte o resultado da condição |

**Aplicação real:** No login de uma rede social, o sistema verifica: `usuário correto E senha correta`. Se uma das duas falhar, o acesso é negado.

---

#### 📌 Tabela Verdade

A **Tabela Verdade** é uma ferramenta que lista todas as combinações possíveis de valores lógicos (Verdadeiro/Falso) para verificar o resultado de uma expressão lógica.

**Tabela do operador AND (E):**

| A | B | A E B |
|---|---|-------|
| V | V | V |
| V | F | F |
| F | V | F |
| F | F | F |

**Tabela do operador OR (OU):**

| A | B | A OU B |
|---|---|--------|
| V | V | V |
| V | F | V |
| F | V | V |
| F | F | F |

---

#### 📌 Estruturas de Controle

São blocos que controlam o **fluxo de execução** de um programa.

**Estrutura condicional (SE/ENTÃO):**
```
SE (condição) ENTÃO
   // bloco executado se verdadeiro
SENÃO
   // bloco executado se falso
FIM SE
```

**Estrutura de repetição (ENQUANTO):**
```
ENQUANTO (condição) FAÇA
   // bloco repetido enquanto a condição for verdadeira
FIM ENQUANTO
```

### 10.2 Prompts Reutilizáveis para Revisão

Estes prompts foram testados e refinados durante o projeto. Podem ser usados diretamente no **NotebookLM**, **Claude** ou qualquer IA generativa para revisar os temas estudados:

---

**🔁 Para revisão de conceito:**
```
Atue como um professor de ensino médio. Explique o conceito de [TEMA] 
de forma didática, usando uma analogia do cotidiano. 
Depois, mostre um exemplo prático simples.
```

**🔁 Para criar exercícios:**
```
Com base no tema [TEMA], crie 3 exercícios de fixação com 
nível crescente de dificuldade (fácil, médio e difícil). 
Inclua o gabarito comentado ao final.
```

**🔁 Para resumo de aula:**
```
Resuma os principais pontos da [NOME DA AULA/CAPÍTULO], 
destacando: (1) conceito central, (2) regra mais importante, 
(3) aplicação prática. Use linguagem simples.
```

**🔁 Para cruzamento de fontes:**
```
Comparando as fontes disponíveis, qual delas explica 
[TEMA] de forma mais acessível para iniciantes? 
Apresente a explicação dessa fonte em camadas: 
primeiro a regra técnica, depois a aplicação prática.
```

**🔁 Para dúvidas em código (Scratch/pseudocódigo):**
```
Preciso entender como [FUNCIONALIDADE] funciona no Scratch. 
Me explique o passo a passo para implementar e mostre 
como isso se traduz em lógica de programação real.
```

## 11. GLOSSÁRIO DE TERMOS TÉCNICOS

| Termo | Definição |
|-------|-----------|
| **Algoritmo** | Sequência de passos lógicos para resolver um problema. |
| **Curadoria** | Processo de selecionar e organizar as melhores informações sobre um assunto. |
| **IA Generativa** | Inteligência capaz de criar conteúdos novos a partir de comandos. |
| **Prompt** | A instrução enviada para a Inteligência Artificial. |
| **Scratch** | Plataforma visual de blocos para aprender programação. |
| **Tabela Verdade** | Ferramenta lógica para verificar se afirmações são verdadeiras ou falsas. |
| **Variável** | Espaço na memória usado para guardar dados que podem mudar. |
| **Operador Lógico** | Símbolo usado para combinar ou inverter condições (E, OU, NÃO). |
| **Fluxograma** | Representação visual dos passos de um algoritmo. |
| **Pseudocódigo** | Forma de escrever algoritmos em linguagem próxima ao humano, antes de codificar. |
